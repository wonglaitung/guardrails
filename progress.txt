# 项目进度记录

## 2026-04-18

### 已完成
- [x] 香港电话识别器 (HK_PHONE_NUMBER)
  - 支持 5-9 开头的 8 位数字
  - 支持带空格格式 (xxxx xxxx)
  - 繁体中文和英文上下文关键词

- [x] 香港身份证识别器 (HK_ID_CARD)
  - 单字母格式: A123456(7)
  - 双字母格式: AB123456(A)
  - 校验码支持数字和字母 A

- [x] 测试文件 test_hk_pii.py
  - 8 个测试模块，全部通过
  - 覆盖边界情况和混合内容

- [x] LLM 安全护栏技术研究
  - 研究 twman.org/cyber/llm-guard 报告
  - 分析主流方案：NeMo, Llama Guard, ShieldGemma, Qwen3Guard
  - 实测 llm-guard，发现中文 PII 支持不足
  - 结论：不集成 llm-guard，保持轻量独立

- [x] LLM-as-a-Judge 概念学习
  - 理解语义检测 vs 规则检测的区别
  - 明确本项目定位：规则检测（精准、快速）
  - 明确扩展方向：可考虑轻量语义检测

### 当前支持的 PII 类型

| 类型 | 格式 | 示例 |
|------|------|------|
| CN_PHONE_NUMBER | 大陆手机 11 位 | 13812345678 |
| CN_ID_CARD | 大陆身份证 18/15 位 | 110101199001011234 |
| CN_BANK_CARD | 银行卡 16-19 位 | 6222021234567890 |
| CN_PASSPORT | 护照 E/G+8位数字 | E12345678 |
| CN_SOCIAL_CREDIT_CODE | 统一社会信用代码 | 91110108MA01234567 |
| CN_LICENSE_PLATE | 车牌号 | 京A12345 |
| EMAIL_ADDRESS | 邮箱 | test@example.com |
| IP_ADDRESS | IPv4 地址 | 192.168.1.1 |
| HK_PHONE_NUMBER | 香港电话 8 位 | 91234567 |
| HK_ID_CARD | 香港身份证 | A123456(7) |

### 技术决策记录

| 决策 | 结论 | 原因 |
|------|------|------|
| 是否集成 llm-guard | 不集成 | 中文PII支持差，体积大 |
| 是否需要 LLM-as-a-Judge | 可选扩展 | 当前规则检测已满足PII需求 |
| 架构方向 | 保持轻量独立 | 精准+快速是核心优势 |
